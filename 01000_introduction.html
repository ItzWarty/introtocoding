Lesson 0: Thinking in Code

Programming is teaching computations and logic to computers given some assumptions:

  1. Computers are dumb.

    Computers follow instructions.

    If you teach a computer incorrectly, then it will perform incorrectly. Garbage In? Garbage Out.

  2. Computers are fast.

    When taught correctly, computers outpace the fastest of humans. Computers are the chess champions of the world for a reason!

  3. Computers are reliable.

    A computer will correctly follow your instructions! You should not assume your logic will be ignored.

Building Blocks

Hardware

At the end of the day, our software runs on physical or virtual hardware. For the purposes of this text, this hardware contains two components:

  • Memory: An infinitely large data-store.
    • Contains Code and Data
  • CPU: Processes memory's operations and data.

Memory

Our code is usually compiled from a human-readable language to machine-readable bits.

Consider the human-readable and contrived machine-readable code that follows:

section .data
  DEFINE done_text "Done!"

section .code
  STORE i, 0
begin: 
  COMPARE i, 5
  JUMPIFEQUAL end
  INCREMENT i
  PUSH i
  CALL console_log
  JUMP begin
end:
  PUSH done_text
  CALL console_log
  EXIT
i = 0;
while (i != 5) {
  i = i + 1;
  console.log(i);
}
console.log("Done!");
Output:
1
2
3
4
5

Which would you rather write? Of course, the first!
